Architecture Summary

CENTRALIZED OFFER MANAGEMENT — Architecture
============================================

Config File (single source of truth):
  /assets/js/offer-config.js
    └── window.MH_OFFERS[]  (array of offer objects)

Engine:
  /assets/js/offer-manager.js
    ├── getActiveOffer()    — filters by enabled + date range, picks highest priority
    ├── injectBanner()      — injects .offer-banner before <nav>, sessionStorage dismiss
    ├── injectNavbar()      — injects .offer-nav-link into .nav-links + .mob-menu
    └── injectPopup()       — delayed modal overlay, sessionStorage dismiss
    └── MHOfferManager.closePopup()  — public API for popup dismiss button

Styles:
  /assets/style.css
    └── .offer-banner, .offer-close, .offer-nav-link, .offer-mob-link,
        .offer-popup-overlay, .offer-popup-card, .offer-popup-cta …

Page Integration (all 226 HTML pages):
  <script src="/assets/js/offer-config.js"></script>
  <script src="/assets/js/offer-manager.js"></script>
  injected before </body> in every page — one-time, never needs editing again.

To run a new campaign:
  1. Add offer object to window.MH_OFFERS in offer-config.js
  2. Set enabled: true, start_datetime, end_datetime
  3. Commit & deploy — zero template edits

To disable:
  Set  enabled: false  in offer-config.js → vanishes site-wide instantly.
